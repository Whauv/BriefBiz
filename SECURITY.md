# Security Policy

## Supported Branch

Security fixes should target `main`.

## Reporting a Vulnerability

If you believe you have found a security issue in BriefBiz, please report it privately to the repository maintainers instead of opening a public GitHub issue.

When reporting, include:

- a short description of the issue
- affected areas or files
- reproduction steps
- expected impact
- any suggested remediation

Please avoid disclosing sensitive details publicly until the issue has been reviewed and a fix is available.

## Scope

This project handles authentication, third-party APIs, and deployment infrastructure. Please treat findings involving secrets, auth flows, request validation, file generation, and worker execution paths as potentially sensitive.
